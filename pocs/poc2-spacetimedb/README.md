# POC 2: SpacetimeDB 2.0 TypeScript Module Connectivity

Proves that SpacetimeDB 2.0 TypeScript modules work end-to-end:

1. TypeScript module deploys and runs on SpacetimeDB 2.0
2. Tables, reducers, subscriptions work
3. Event tables work for transient events
4. Scheduled reducers work (game tick at 1-second interval)
5. Client can connect, subscribe, call reducers, and see updates

## Project Structure

```
poc2-spacetimedb/
  server/
    spacetime.json            # SpacetimeDB project config
    spacetime.local.json      # Local database name
    package.json              # Dependencies: spacetimedb ^2.0.2
    tsconfig.json             # TypeScript config
    spacetimedb/
      package.json            # Module package
      tsconfig.json           # Module TS config
      src/
        schema.ts             # Tables, constants, schema export
        index.ts              # Reducers, lifecycle hooks, re-exports schema
  client/
    index.html                # Browser client with canvas + event log
    src/module_bindings/      # Generated by spacetime generate (not checked in)
  verify.sh                   # CLI-based verification script
```

## Server Module

**Tables:**
- `player` -- one row per connected identity (name, position, health, online)
- `creature` -- spawned creatures with movement targets (owner, position, speed)
- `event_combat` -- transient event table for combat events (not persisted)
- `tick_schedule` -- scheduled table that triggers tick_world every second

**Reducers:**
- `init` -- seeds the tick schedule on first publish
- `onConnect` / `onDisconnect` -- manage player records
- `spawn_creature(x, y)` -- spawn a creature at coordinates
- `move_creature(creatureId, targetX, targetY)` -- set creature movement target
- `tick_world` -- scheduled every 1s; moves creatures, checks combat proximity

## Setup

```bash
# 1. Start local SpacetimeDB (if not already running)
spacetime start

# 2. Install dependencies
cd server/spacetimedb && npm install && cd ../..

# 3. Build and publish (verify.sh does both)
./verify.sh

# Or manually:
cd server && spacetime build && spacetime publish poc2 --clear-database -y -s local && cd ..
```

## CLI Verification (no client needed)

```bash
# All commands use -s local for local SpacetimeDB

# Spawn a creature
spacetime call poc2 spawn_creature 100.0 150.0 -s local

# Query tables
spacetime sql poc2 'SELECT * FROM player' -s local
spacetime sql poc2 'SELECT * FROM creature' -s local
spacetime sql poc2 'SELECT * FROM tick_schedule' -s local

# Move a creature (replace 1 with actual creature ID)
spacetime call poc2 move_creature 1 300.0 300.0 -s local

# Watch logs (tick_world runs every second)
spacetime logs poc2 -s local -f

# Subscribe to real-time updates
spacetime subscribe poc2 'SELECT * FROM creature' -s local
```

## Browser Client

```bash
# 1. Generate client bindings
cd server && spacetime generate --lang typescript --out-dir ../client/src/module_bindings -s local && cd ..

# 2. Serve the client
cd client && python3 -m http.server 8080

# 3. Open http://localhost:8080 in browser
```

The browser client shows:
- 2D canvas with creatures as colored circles
- Left-click to spawn a creature
- Right-click a creature to select, right-click elsewhere to move it
- Connection status, player/creature counts
- Event log showing connections, spawns, movements, and combat events

**Note:** The browser client requires generated bindings to function. Without them,
it falls back to displaying CLI commands for verification.

## What This Proves

| Objective | How Verified |
|-----------|-------------|
| TS module deploys | `spacetime publish` succeeds, `spacetime sql` responds |
| Tables + reducers | `spawn_creature` inserts row, queryable via SQL |
| Event tables | `event_combat` table exists with `event: true`, queryable |
| Scheduled reducers | Creature position changes after ticks (measured x > 11 after spawning at x=10) |
